# SmartPlay Watch (Wear OS companion)

Standalone Wear OS app that captures **wrist-IMU motion during a golf
swing** (accelerometer + gyroscope at ~200 Hz), segments it on-device,
and streams a per-swing summary to the SmartPlay phone app over the
**Wearable Data Layer**. On the phone, the summary lands in
`store/watchStore.ts` via `recordSwing()` and promotes club speed / tempo
/ smash factor to the truth-grade `'watch'` source tier in
`services/swingMetricsService.ts`.

## Why this is a separate project (not an Expo module)

The phone app is an Expo **managed** project — its `android/` tree is
generated by `expo prebuild` at EAS build time. Wiring a Wear module into
that generated build would mean mutating the auto-generated
`settings.gradle`, which is exactly the kind of change that can break the
**phone APK build**. To guarantee *no regression to the working phone
APK*, this watch app is a fully independent Gradle project. It is never
referenced by the phone build and cannot affect it.

## ⚠️ Pairing requirement (the one real gotcha)

The Wearable Data Layer only auto-associates two apps when they share:

1. **The same `applicationId`** — already set to `com.smartplaycaddie.app`
   in `app/build.gradle` (matches the phone). Do not change it.
2. **The same signing key** — the watch APK must be signed with the SAME
   keystore as the phone APK. The phone's key is managed by EAS.

To get the EAS signing key for local signing:

```bash
# From the phone project root (/Users/timothyg/smartplay):
eas credentials            # Android → download/export the keystore
```

Then point this project's release signing at that keystore (add a
`signingConfigs` block to `app/build.gradle`), or sign the built APK with
`apksigner` using the exported keystore. For quick bench testing you can
also sign BOTH the phone debug build and this app with the same debug
keystore (`~/.android/debug.keystore`).

## Build & install

```bash
cd wear-os-app
./gradlew :app:assembleDebug          # (run `gradle wrapper` once if no wrapper)
# Install to a watch connected via adb (Wi-Fi debugging or USB):
adb -s <watch-id> install -r app/build/outputs/apk/debug/app-debug.apk
```

Open **SmartPlay Watch** on the watch, tap **Start**, and make swings. A
foreground notification ("Capturing swing motion") confirms capture is
live; it keeps running while the screen is off mid-shot.

## What it measures (honesty)

Measured directly by the IMU: backswing/downswing timing, tempo ratio,
peak wrist angular velocity, peak/impact acceleration.
**Derived estimate:** `clubHeadSpeedEst = peakWristAngularVel × clubRadius`
(single-pendulum approximation; the radius is a calibratable assumption,
not a launch-monitor reading). See `SwingDetector.kt` header for the full
measured-vs-derived breakdown.

## Data Layer contract (phone ↔ watch)

- Path `"/smartplay/swing"` — JSON swing summary (fields map 1:1 to the
  phone's `SwingMetrics`). Received by `android-native/WearSwingBridgeModule.kt`.
- Path `"/smartplay/hello"` — presence ping when capture starts; the phone
  marks the watch connected.

# Navigation / screen graph (expo-router) — /Users/timothyg/smartplay/app

> Generated by QA audit 2026-07-21.

**Summary:** The app uses expo-router file-based routing with a single root Stack (app/_layout.tsx) wrapping one bottom-tab group (app/(tabs)) plus ~80 stack/modal screens and several nested stacks (cage, cage-review). app/index.tsx is the routing brain: it gates the onboarding chain (intro-video → permissions → welcome → greeting → caddie) on store hydration + a terms-consent flag, then lands on the Caddie tab. There is NO auth/login gate — a subscription trial lifecycle exists but is globally short-circuited (SUBSCRIPTIONS_ENABLED=false grants everyone lifetime), so every screen is reachable without login. ~15 owner/dev debug routes are centrally gated in _layout.tsx via a DEBUG_ROUTES redirect (non-owner + non-__DEV__ → bounced to Caddie). I found two effectively-dead route trees: the entire app/cage/* group (retired "Cage Mode", no inbound navigation to cage/index) and app/course/[course_id] (registered but nothing pushes to it — deep-link only), plus app/dev/CourseTruth which is reachable only by deep link/dev.

# SmartPlay Caddie — Navigation / Screen Graph

Router: **expo-router v6**, file-based, `scheme: smartplay`, entry `expo-router/entry`.
Root file: `/Users/timothyg/smartplay/app/_layout.tsx` — one `<Stack>` with `headerShown:false`, `animation:'fade'` defaults, wrapped in ErrorBoundary + SmartVision/KevinPresence/SafeArea/Theme providers. Global overlays (tools menu, caddie mic, toasts, GPS dot, rest mode, banners) are mounted here, above the Stack, so they ride on every screen.

`app/api/*+api.ts` are **server API route handlers** (brain, voice, transcribe, kevin, cage-review, etc.), not screens — excluded from the screen graph.

## 1. Entry / routing brain

### `app/index.tsx` (route `/`) — the redirect hub
Blocks render until BOTH `playerProfileStore` and `settingsStore` finish AsyncStorage hydration (`if (!hydrated) return null`), then redirects in this priority order:

1. `!tutorialsSeen.intro_video` → `/intro-video`
2. `!tutorialsSeen.core_permissions_requested` → `/permissions`
3. `!isDone` (no onboarding/setup) → `/welcome`
4. `!hasAcceptedTerms && !hasName` → `/welcome`  *(gate is on `termsAcceptedAt`, NOT `first_opened_at`, because the boot trial-lifecycle stamps `first_opened_at` during hydration)*
5. `kevinGreetingEnabled && !greetingShownThisProcess` → `/greeting` (once per cold launch)
6. else → `/(tabs)/caddie`

### Onboarding chain (all `gestureEnabled:false`, fade)
`/intro-video` → `router.replace('/')` → re-evaluates → `/permissions` → `router.replace('/')` → `/welcome` (accept terms sets `termsAcceptedAt`, captures name+caddie) → `router.replace('/(tabs)/caddie')`, or "Quick tour" → `/quick-start`. Greeting (`/greeting`) plays persona voice, warms the backend, then `router.replace('/(tabs)/caddie')`.

- **No auth/login screen exists.** Removed long ago (comments at `_layout.tsx:1018`). Owner identity is inferred from email allow-list, not a login.
- **Trial/paywall is inert:** `_layout.tsx` trial lifecycle grants everyone `lifetime` when `SUBSCRIPTIONS_ENABLED === false`. `/paywall` is a registered modal but only fired by a deferred-paywall guard + debug screens; in practice never blocks.
- **Every non-debug screen is reachable without onboarding** via deep link (nothing re-checks consent once you're past `index.tsx`); the onboarding gate only governs the cold-launch flow from `/`.

### `app/+not-found.tsx`
Catch-all (needed for web static export). Native flows shouldn't reach it; renders a `<Link href="/(tabs)/caddie">`.

## Route-kind legend
- **tab**: child of `app/(tabs)` bottom-tab navigator (5 tabs).
- **modal**: `presentation:'modal'` (only `/paywall`) or `animation:'slide_from_bottom'` sheet-style (`/swinglab/upload`, `/swinglab/space-scan`, `/swinglab/tutorial-upload`, `/quick-start`, `/jukebox`, `/smartfinder`, `/lie-analysis`, `/recap/[round_id]`, most debug screens).
- **stack**: default `slide_from_right`/`fade` push screen.

## Known risks
- app/course/[course_id].tsx is registered but has no inbound navigation edge (Play tab routes to /course-layout instead) — orphaned/deep-link only; confirm intended before relying on it.
- The entire app/cage/* group appears dead: cage/index.tsx (/cage) has no inbound router.push and Cage Mode was retired/merged into Smart Motion (per code comments). The cage-review sub-tree is only reachable through the owner-gated /cage-debug screen.
- No auth/login exists and the subscription/paywall system is globally short-circuited (SUBSCRIPTIONS_ENABLED=false grants lifetime), so /paywall is effectively unreachable in normal flow and every screen is reachable without payment.
- Onboarding gating lives entirely in app/index.tsx and only runs on the cold-launch flow from '/'; deep links (scheme smartplay) bypass the consent/onboarding gate for any registered screen.
- Debug-route protection is a single useEffect redirect in _layout.tsx keyed on pathname + owner email; it depends on usePathname matching the exact strings in DEBUG_ROUTES (e.g. query-string or trailing-slash variants could slip past the central gate, relying on per-screen belt-and-suspenders checks).
- /owner-logs (Issue Log) is intentionally excluded from DEBUG_ROUTES so all beta testers can file bugs; only its AI-triage control is owner-gated — worth confirming nothing sensitive is exposed there.
- Two overlapping tutorial screens exist (root /tutorials and /swinglab/tutorials); the root one has thin inbound wiring and may be a partial dead/duplicate surface.
- /dev/CourseTruth and /harness have no UI navigation edges (only appear in the gate list) — reachable only via deep link in dev.
